# 靈修冒險｜多 Agent 角色系統

> 最後更新：2026-05-09
> 維護：PM 視窗統籌

## 系統概觀

靈修冒險專案採「5 個策略視窗 + Claude Code 執行者」架構。
策略視窗在 Claude.ai 各自獨立運作，Claude Code 是唯一執行者。

## 角色清單

| 視窗 | 角色檔案 | 主要職責 |
|------|---------|---------|
| 🎯 PM 總指揮 | `pm-strategist.md` | 戰略決策、跨視窗統籌、進版時機 |
| 💻 開發協調 | `development-coordinator.md` | Phase 切分、Bug 管理、寫 Claude Code 指令 |
| 🎨 美術協調 | `art-director.md` | 視覺 brief、設計整合、design-system 維護 |
| 📊 數據分析 | `data-analyst.md` | 玩家洞察、KPI 追蹤、月報 |
| 📝 內容生產 | `content-creator.md` | 靈修內容、和合本驗證、審查清單 |

## 進度卡共通欄位（最低限度）

每個角色的進度卡至少包含：

1. **更新時間**（年-月-日 + 視窗來源）
2. **當前任務**（標題 + 狀態 + 工程量）
3. **重啟指令**（給未來新視窗的「我要繼續 X」）

其他欄位由各視窗依自身性質決定。

## 跨視窗傳遞規則

**重要結論用貼文，背景脈絡用搜尋。**

| 情境 | 傳遞方式 |
|------|---------|
| PM 拍板結論 | 📋 貼文(整段複製到目標視窗) |
| 跨視窗最終決議 | 📋 貼文 |
| 要對方執行的指令 | 📋 貼文 |
| 參考歷史脈絡 | 🔍 conversation_search |
| 背景理解 | 🔍 conversation_search |

理由：conversation_search 索引延遲不可預測，重要結論不能賭。

## 模糊地帶處理

當任務性質不確定時：
- 寧可多問 PM 一次，不要默默做出戰略級決定
- PM 視窗使用 conversation_search 跨視窗讀取脈絡，整合判斷後給出明確歸屬
