## ⚠️ 第六部分：第 2 任 PM 犯的錯（新 PM 引以為戒）

第 2 任 PM 在 W21 期間犯了幾個錯，James 每次都溫和糾正。記下來讓你別重蹈：

1. **越界寫 Claude Code dispatch prompt**（5/18）
   - 當 James 問「能不能用 subagent」，PM 答到「判斷適合與否 + 架構注意事項」就該停，卻一路把完整 dispatch prompt 寫完。
   - 紀律：PM 給方向、不寫指令。指令找對應視窗。答到「架構注意事項」就停。

2. **dispatch 前沒先 view 既有檔案**（5/18）
   - 派 LEARNING.md 補寫任務時、憑記憶下 Acceptance Criteria，結果叫 Claude Code 做「前一個 commit 已經做過的事」。
   - 紀律：涉及既有檔案補寫的任務，prompt 必須含「前置動作：先 view 既有結構、確認不重複」。

3. **時間感漂移**（5/21）
   - James 凌晨告知一次時間，PM 把那個時點當錨點、後續 10+ 訊息全推測「凌晨 3 點」，實際已是當晚 22:40，誤差 5 小時。
   - 紀律：任何時間判斷（該不該睡、死線剩幾天）先查時間工具，不推算。本視窗後期甚至發現該工具不一定可用 —— 不可用時就誠實問 James、不假裝知道。

4. **派出帶過時資訊的指令**（5/24 前）
   - 派了一個 PHP1-4 部署指令、裡面帶過時假設（BOOKS 沒補、3D 狀態誤標），被開發協調校正三處。PM 甚至不記得自己派過這個指令 = context 已被壓縮。
   - 紀律：context 過載是會發生的。出現「不記得自己做過的事」就是該交班的紅燈，別硬撐。

**共通根因**：context 過載 + 靠殘缺記憶決策。新 PM 的解法 —— 報進度前先確認來源（commit 鏈 / 開發協調 / 問 James），別信自己的記憶。
