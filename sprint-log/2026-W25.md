## 九、PM 觀察 / LEARNING（這幾天累積）

- **「上 main 前先 git 查證」救過事故**：TIM2 無日期 gating、會提早曝光 → 治本時改 cherry-pick 排除
- **跨視窗歸因錯誤被擋下**：內容視窗「疑似 L31」是未查證歸因，開發協調看原文擋下、避免改壞核心紀律
- **PM 優先級偏差被 CC 校正**：急推 HEB（軟 deadline）卻漏了 TIM2 上線（6/30 硬錨點、已備好待釋出）
- **merge vs cherry-pick 被開發協調修正**：PM 說「merge 收分叉」，但分叉已被 0301ad9 reverse merge 收掉、且 merge 會把內部文件弄髒玩家版 main → 改 cherry-pick
- **共通教訓**：PM 憑記憶/慣性的判斷，靠下游視窗查證現實擋下——多視窗互相 check 是健康的，但 PM 自己也要更常「先查證再下結論」
- ⚠️ **cherry-pick 累積分叉**：每次 cherry-pick 上 main 都讓 dev↔main 分叉更複雜，git 清理 sprint 時要整體盤一次
